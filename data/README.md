# Инструкция к Google-таблице для пульта Сушиселл

Эта таблица является источником данных для пульта. Приложение читает из неё сотрудников, роли, метрики и уже сохранённые отчёты. Чтобы интеграция не сломалась, меняйте только содержимое разрешённых ячеек и не трогайте техническую структуру листов.

## Какие листы использует пульт

Пульт читает данные только из этих листов:

1. `Инфо` — список сотрудников, их ролей и руководителей.
2. `HR` — метрики HR-направления.
3. `ОКС` — метрики ОКС.
4. `Франшиза` — метрики франшизы.
5. `Финансы` — метрики финансов.
6. `Операционный` — операционные метрики.
7. `Производство` — производственные метрики.
8. `Маркетинг` — метрики маркетинга.
9. `Данные` — технический лист, куда пульт записывает заполненные отчёты.

Если одного из листов с метриками нет, пульт просто пропустит его. Если переименовать лист, данные с него перестанут попадать в пульт.

## Что можно делать

### На листе `Инфо`

Можно:

- добавлять новых сотрудников новыми строками;
- исправлять отдел в колонке A;
- исправлять подотдел в колонке B — по нему метрики на дашборде группируются ступенчато;
- исправлять ФИО в колонке C;
- менять роль сотрудника в колонке D;
- менять роль руководителя в колонке E; пульт строит дашборд строго по цепочке из колонок D «Роль» и E «Руководитель», поэтому директор видит личные метрики каждого руководителя и метрики всех сотрудников ниже по этой цепочке;
- добавлять одному сотруднику несколько ролей через `/`, `,`, `;`, `|`, `и` или `или`.

Формат колонок:

| Колонка | Что хранится | Обязательно |
| --- | --- | --- |
| A | Отдел | Да |
| B | Подотдел | Нет |
| C | ФИО сотрудника | Да |
| D | Роль / должность сотрудника | Да |
| E | Роль руководителя | Нет |

### На листах с метриками

Можно:

- добавлять новые метрики новыми строками;
- редактировать текст метрики, описание, цель, дедлайн и ответственного;
- оставлять пустыми необязательные поля;
- использовать ссылки в описании — пульт сделает их кликабельными;
- указывать несколько ответственных ролей через `/`, `,`, `;`, `|`, `и` или `или`.

Формат колонок на листах `HR`, `ОКС`, `Франшиза`, `Финансы`, `Операционный`, `Производство`, `Маркетинг`:

| Колонка | Что хранится | Обязательно | Пример |
| --- | --- | --- | --- |
| A | Периодичность | Да | `ежедневно`, `еженедельно`, `ежемесячно`, `ежеквартально` |
| B | Название метрики | Да | `Проверить отчёт по сменам` |
| C | Описание / инструкция | Нет | `Проверить документ по ссылке ...` |
| D | Цель | Нет | `Нет просроченных задач` |
| F | Должность ответственного | Да | `HRD` |
| G | Срок сдачи | Нет | `18:00`, `пятница`, `5 число` |
| I | Классификация метрики | Нет | `Проверено`, `Ввод числа`, `Ввод процента` |
| J | Руководитель для дашборда | Нет | `Операционный директор` |

### На листе `Данные`

Лист `Данные` заполняется автоматически из пульта. Его можно просматривать, фильтровать и использовать для сводных таблиц, но вручную редактировать нежелательно.

Формат колонок:

| Колонка | Что хранится |
| --- | --- |
| A | Дата отчёта |
| B | ФИО сотрудника |
| C | Метрика |
| D | Значение |
| E | Комментарий |

## Что нельзя делать

Не делайте эти действия без согласования с ответственным за пульт:

- не переименовывайте листы `Инфо`, `Данные`, `HR`, `ОКС`, `Франшиза`, `Финансы`, `Операционный`, `Производство`, `Маркетинг`;
- не удаляйте эти листы;
- не меняйте порядок и назначение колонок A, B, C, D, F, G, I, J на листах с метриками;
- не меняйте порядок и назначение колонок A, B, C, D, E на листе `Инфо`;
- не меняйте порядок и назначение колонок A–E на листе `Данные`;
- не удаляйте первую строку с заголовками;
- не объединяйте ячейки в рабочих колонках;
- не добавляйте дополнительные строки заголовков над таблицей;
- не вставляйте формулы, которые возвращают разные типы данных в обязательные колонки;
- не редактируйте и не удаляйте строки на листе `Данные`, если не уверены, что отчёты нужно перезаписать;
- не меняйте URL опубликованного Apps Script без обновления `window.PULT_DATA_URL` в приложении.

## Правила заполнения

- ФИО в пульте ищется по точному тексту из листа `Инфо`, поэтому пишите ФИО одинаково во всех местах.
- Роль сотрудника из `Инфо` должна совпадать с ролью ответственного в колонке F на листах с метриками.
- Если метрика должна показываться нескольким ролям, перечислите роли в колонке F через разделитель.
- Если сотрудник совмещает несколько ролей, перечислите роли на листе `Инфо` в колонке D.
- Для числовой метрики укажите в колонке I значение `Ввод числа`.
- Для обычной проверки укажите в колонке I значение `Проверено` или оставьте колонку пустой.
- Время в колонке G считается московским временем (`11:00` или `11:00 МСК`) и в пульте автоматически показывается в часовом поясе устройства сотрудника; если время уже задано по Красноярску, подпишите его как `15:00 Крск`.
- Поддерживаемые периодичности: `ежедневно`, `еженедельно`, `ежемесячно`, `ежеквартально`.
- Дату отчёта, отдел и ФИО сотрудник выбирает в пульте перед тем, как увидеть список метрик.

## Как подключается таблица к пульту

Пульт ожидает JSON в таком формате:

```json
{
  "infoRows": [
    { "department": "ОКС", "subdepartment": "Забота", "fullName": "Фамилия Имя Отчество", "role": "Роль", "managerRole": "Роль руководителя" }
  ],
  "metricSheets": [
    {
      "name": "HR",
      "rows": [
        {
          "frequency": "ежедневно",
          "metric": "Название метрики",
          "description": "Описание",
          "goal": "Цель",
          "deadline": "18:00",
          "role": "Ответственная роль",
          "managerRole": "Руководитель для дашборда",
          "classification": "Проверено"
        }
      ]
    }
  ]
}
```

Обычно вручную JSON делать не нужно: Google Apps Script из файла `data/google-apps-script.js` сам читает таблицу и отдаёт данные пульту.

## Публикация через Google Apps Script

1. Откройте Google Sheets → `Расширения` → `Apps Script`.
2. Скопируйте код из `data/google-apps-script.js` в `Code.gs`.
3. Проверьте в `CONFIG`, что список `metricSheetNames` содержит нужные листы: `HR`, `ОКС`, `Франшиза`, `Финансы`, `Операционный`, `Производство`, `Маркетинг`.
4. Нажмите `Deploy` → `New deployment` → тип `Web app`.
5. Выберите `Execute as: Me` и `Who has access: Anyone with the link`.
6. Скопируйте URL веб-приложения и вставьте его в `src/config.js` в значение `window.PULT_DATA_URL`.
7. После изменения URL или фронтенда увеличьте версию в `package.json`, `src/version.js` и query-параметры ресурсов в `index.html`, чтобы браузер не показывал старые файлы из кеша.

## Если данные не появились в пульте

Проверьте по порядку:

1. Лист называется ровно как в списке выше, без лишних пробелов.
2. На листе есть строка заголовков и хотя бы одна строка данных ниже.
3. В строке метрики заполнены колонки A, B и F.
4. В `Инфо` заполнены колонки A, C и D для выбранного сотрудника; колонка B заполняется для группировки на дашборде.
5. Роль сотрудника из `Инфо` совпадает с ролью в колонке F метрики.
6. Apps Script опубликован как Web app с доступом `Anyone with the link`.
7. URL Web app указан в `src/config.js`.
